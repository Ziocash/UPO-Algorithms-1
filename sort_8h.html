Sorts the given array according to the insertion sort algorithm.

Parameters
basePointer to the start of the input array.
nNumber of elements in the input array.
sizeThe size (in bytes) of each element of the array.
cmpPointer to the comparison function used to sort the array in ascending order. The comparison function is called with two arguments that point to the objects being compared and must return an interger less than, equal to, or greater than zero if the first argument is considered to be respectively less than, equal to, or greater than the second.

Insertion sort uses compares and exchanges to sort a randomly ordered array with distinct keys, on the average. The worst case is compares and exchanges, and the best case is compares and exchanges. The time complexity of insertion sort is in the worst case.

Sorts the given array according to the merge sort algorithm.

Parameters
basePointer to the start of the input array.
nNumber of elements in the input array.
sizeThe size (in bytes) of each element of the array.
cmpPointer to the comparison function used to sort the array in ascending order. The comparison function is called with two arguments that point to the objects being compared and must return an interger less than, equal to, or greater than zero if the first argument is considered to be respectively less than, equal to, or greater than the second.

Merge sort performs between $\frac{1}{2} n \log n$ and $n \log n$ compares and at most $6n \log n$ array accesses.
